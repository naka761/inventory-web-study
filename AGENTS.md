# Repository Instructions

- `docs/` 配下の Markdown は、日本語の初心者向け教科書として作成する。
- 「何をしているか」だけでなく、「なぜ必要か」「何を防ぐか」「誰と誰をつなぐか」を説明する。
- 実際の `inventory-web-study` のコードと設定を確認してから書く。
- 推測は断定せず、環境依存ならその旨を書く。
- 表、コードブロック、テキスト図を適切に使う。
- 不必要な重複説明を避ける。
- ユーザーから明示されない限り、既存の Java、JSP、`pom.xml` は変更しない。
- ドキュメント作成依頼では、作成後に自分で内容と差分を確認する。
- 問題がなければ、指定されたドキュメントだけを commit して GitHub へ push する。
- push 前に DB パスワード、トークン、秘密情報が含まれていないことを確認する。
